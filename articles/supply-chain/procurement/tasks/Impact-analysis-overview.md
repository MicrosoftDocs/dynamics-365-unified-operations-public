The Impact Analysis capability within the Procurement Agent helps procurement teams save time on analyzing if each change communicated by suppliers has downstream impact, and helps them focus on change requests from suppliers that matter. It automatically evaluates the downstream impact of a supplier-communicated change on orders and inventory levels, so purchasers can immediately distinguish between changes that are safe to accept, and those requiring further action.

Purchasing teams regularly receive change requests from suppliers, including delivery delays, quantity reductions, and cancellations. Each change to a purchase order can potentially affect inventory levels, production schedules, internal transfers, and customer commitments. For changes with downstream impact on orders, or ones that cause inventory levels to go below certain thresholds, purchasers often push back on the supplier to get better terms such as expediting production or transportation resulting in an earlier delivery date or increased quantity delivered or splitting the delivery. And if they must accept a change with impact, they need to determine which internal stakeholders to inform based on what will be impacted. Determining whether a supplier change is safe to accept or warrants action today is a manual and time-consuming process of digging into planning data.
This enables purchasers to save manual effort and to prioritize their time on changes that truly affect service levels, inventory, or operations. It helps to improve responsiveness, increases the likelihood of protecting production schedules and customer commitments, and keeps the purchase order up-to-date. 

## Review the impact of purchase order changes from vendors ##

With Impact Analysis, purchasers can quickly see if a change request from suppliers has an impact on downstream orders and on inventory levels. 

In most cases, purchasers will not want to spend time on changes to purchase orders without downstream impact. The agent gives purchasers the confidence and ability to apply these changes right away and move on. But when a change has downstream impact, purchasers need to quickly understand the details, consequences, and decide how to respond. 

Based on the impact analysis, purchasers can decide whether to follow up with the supplier. If, for example, the impact is very significant to production and/or sales orders and there is enough time to do so, the purchaser may look for alternatives and have conversations with the supplier and internal stakeholders to see if anything can be done. Based on the details in the impact analysis, they know who to contact.

In situations where the purchaser decides there is no viable mitigation, or when the change does not have impact, the agent helps apply the changes directly to the purchase order from the same user experience, eliminating the need to manually locate and update the order.
See [Review impact of purchase order changes from vendors](tasks/impact-analysis-review-changes.md).

## Simulate if purchase order changes have impact ##

The Impact simulation capability allows purchasers and planners to test the impact of changes not directly communicated through e-mails via the Supplier Communications Agent or the Vendor Collaboration Module.

Some key use cases for this simulation include: 
- Testing and getting a feel for the impact analysis feature before using it on live supplier change requests.
- Proactively testing potential scenarios for at-risk or high-priority purchase orders. This could be purchase orders for which you have received no reply yet.
- Evaluating if supplier proposals to new dates and/or quantities that are communicated after following up solve the impact. This can be done by comparing the output of a simulation to the impact analysis of the communicated change.
- Evaluating changes received and reviewed in external channels that do not automatically trigger an impact analysis.
See [Simulate if purchase order changes have impact](impact-analysis-simulation.md).
